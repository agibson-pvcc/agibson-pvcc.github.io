All About Github

  • Q. What is a version control system?
  • A. Version control systems intelligently track changes in files. It is most helpful when a group of people are all making changes to the same files.

  • Q. What is Git?
  • A. Git is a version control system for tracking changes in code.

  • Q. What is GitHub?
  • A. GitHub is a cloud-based platform where you can store, share, and work together with others to write code.

  • Q. What is the difference between Git and Github?
  • A. GitHub is the hosting service to store your code but Git is the tool used for managing and tracking changes in the code AKA version control.

  • Q. Who started Github and how was it started?
  • A. GitHub was founded by Chris Wanstrath, P. J. Hyett, Tom Preston-Werner and Scott Chacon.

  • Q. What company owns it now?
  • A. Microsoft purchased GitHub in 2018 for $7.5 billion U.S. dollars.

  • Q. How much does a GitHub account cost?
  • A. GitHub is free for their basic service plan with limitations. They also offer upgraded plans that costs $4 and $21 per month.

  • Q. What is the octocat?
  • A. The octocat is GitHub's mascot! Created by Cameron McEfee.


Git / GitHub Terminology

  1. Repository - Is the most basic element of GitHub. Essentially just the folder that holds all of your files and documents.
  2. Commit - Is a revision made to one of your files with a user-typed message describing the changes made.
  3. Fork - Is a personal copy of another persons repository. Changes made to this will not effect the repository that was copied.
  4. Push - To send your committed changes to a remote repository.
  5. Pull Requests - Are proposed changes or ideas made by one user and peer-reviewed for acceptance or rejection.
  6. Workflows - Is an automated process that is configurable by the user. It will perform certain actions when triggered by particular events that it is told to look out for.
  7. Issues - Are suggested improvements made by repository collaborators. Typically a discussion thread associated with the proposals.
  8. Raw Button - Used to view and/or copy the raw content without any styling.
  9. Blame Button - Used to view all modifications made to a file with information detailing who made the change, what the change was, and when it was made.
